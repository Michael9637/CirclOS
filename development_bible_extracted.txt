SECTION 1 — WHAT IS CIRCLOS
CirclOS is a dual-product B2B SaaS platform solving two urgent problems for Austrian manufacturing SMEs simultaneously: (1) costly industrial waste disposal, and (2) EU anti-greenwashing compliance (ECGT Directive 2024/825, enforcement from September 2026).
The core insight — one sentence:
The transaction data generated by the waste exchange IS the evidence base required by EU anti-greenwashing law. Each product makes the other more valuable.
Four product layers:
L1 — Waste Exchange (free tier): Companies list industrial byproduct streams. Free to join. This is the acquisition engine.
L2 — AI Matching Engine (paid, €49-129/mo): ML matching by material type, geography, volume. Every confirmed transaction generates an immutable evidence record in the database.
L3 — Compliance Dashboard (premium, €79-299/mo): Web crawler scans company website. German BERT NLP model identifies environmental claims. Claims classified against ECGT Article 6 blacklist. Evidence auto-linked to L2 transaction records. PDF compliance certificate exported.
L4 — Regulatory Reporting API (enterprise): Aggregated, anonymised data sold to large companies (500+ employees) for CSRD Scope 3 supply chain reporting. Year 2+ product — do NOT build in MVP.
Target market (Carinthia beachhead):
Green Tech Valley cluster: 260+ companies in Carinthia & Styria
Wood processors generating 5-50 tonnes of offcuts monthly
Silicon Alps electronics manufacturers (Infineon Villach, SAL, Lam Research)
43,000+ Austrian SMEs addressable via WKO distribution
Key regulatory hooks:
EU ECGT Directive 2024/825: bans 'eco-friendly', 'sustainable', 'green' without verifiable proof. Enforcement: September 2026 in Austria.
Austrian AWG 2002: governs all waste handling. Distinguishes Abfall (waste, licensed buyer required) vs Sekundärrohstoff (secondary raw material, free trade).
CSRD Omnibus (2025): raised threshold to 1,000+ employees. SMEs are now EXEMPT from mandatory CSRD — but large companies still need their suppliers' waste data for Scope 3 reporting.
SECTION 2 — MVP SCOPE (WHAT WE ARE BUILDING)
The MVP covers Layers 1, 2, and 3 in demo quality. 'Demo quality' means: a jury member can log in, use every feature, and see real data flowing. It does NOT mean production-ready scale or security hardening.
IN SCOPE for 3-week MVP:
Company registration and login (email/password via Supabase Auth)
Waste listing form: material type, volume (kg), legal classification, description, location
AI matching: submit a listing → see ranked matches from other listings using vector similarity
Evidence record: when a match is confirmed by both parties → immutable record written to database
Compliance scanner: enter a URL → scrape text → classify green claims → show results with ECGT status
Basic dashboard: company profile, my listings, my matches, my evidence records
Deployment: live on internet (not just localhost) for jury demo
OUT OF SCOPE for MVP (do not build, do not attempt):
Payments / Stripe integration — show pricing page but no real billing
L4 Regulatory Reporting API
PDF certificate export (show the data on screen — PDF generation in Week 4+ if time permits)
Email notifications
Mobile-responsive design beyond basic usability
Fine-tuned BERT model — use base gbert via HuggingFace Inference API
LightGBM ranking — use cosine similarity from sentence-transformers only

SECTION 8 — COMMON ERRORS AND INSTANT FIXES
ERROR: CORS error in browser console
Cause: Your FastAPI backend is blocking requests from your React frontend.
Fix: In main.py, find the CORSMiddleware line. Make sure allow_origins=["*"] — the asterisk means allow all. If the line does not exist, add:from fastapi.middleware.cors import CORSMiddlewareapp.add_middleware(CORSMiddleware, allow_origins=["*"], allow_methods=["*"], allow_headers=["*"])
ERROR: ModuleNotFoundError: No module named X
Cause: A Python package is not installed.
Fix: In terminal, run: pip install XThen restart uvicorn. Example: pip install sentence-transformers
ERROR: supabase.exceptions.APIError: relation does not exist
Cause: The table name in your code does not match the table name in Supabase.
Fix: Go to Supabase → Table Editor and check the exact table name (case sensitive). Update your code to match exactly.
ERROR: TypeError: Cannot read properties of undefined
Cause: React is trying to display data before it has loaded from the API.
Fix: Add a loading check before rendering: if (!data) return <p>Loading...</p>; — put this at the top of your component before the return statement.
ERROR: uvicorn: command not found
Cause: uvicorn was installed but your terminal cannot find it.
Fix: Run: python -m uvicorn main:app --reload (use python -m prefix). This always works regardless of PATH.
ERROR: Railway deploy fails
Cause: Railway cannot find your app's start command.
Fix: Add a file called Procfile (no extension) in your backend/ folder with this one line:web: uvicorn main:app --host 0.0.0.0 --port $PORT
ERROR: HuggingFace API returns 503
Cause: The model is loading (first request takes 20-30 seconds on free tier).
Fix: Wait 30 seconds and try again. If it keeps failing, the free tier may be rate limited. Fallback: use a simple keyword matching approach instead of the API call for the demo.
ERROR: Vite dev server: Failed to fetch
Cause: Frontend is trying to call localhost:8000 but backend is not running.
Fix: Open a second terminal window, navigate to backend/, and run: python -m uvicorn main:app --reload. Both servers must run simultaneously.

Answers to the three jury questions you will definitely be asked:
Q: Does CirclOS need the €300K-€500K seed round to launch?
No. Service 1 — the waste exchange with AI matching — is fully funded by the KWF grant and AAU salaries. The seed round comes after Demo Day, after we already have paying customers and proven MRR. It is for speed of scaling, not for survival.
Q: Does ECGT apply to all SMEs or just B2C companies?
ECGT's strictest provisions apply directly to B2C-facing companies — those using green claims in consumer marketing. For B2B-only SMEs, the driver is indirect: their large customers who are subject to CSRD Scope 3 reporting will require verified waste transaction data from their supply chain. Both paths lead to CirclOS, but we validate the B2C vs B2B split as a Week 1 priority before building compliance urgency into sales messaging.
Q: Is the CirclOS compliance certificate legally valid?
At launch, it is a good-faith documentation record — timestamped, evidence-backed, and linked to immutable transaction records. ECGT does not require a government-approved certifier; it requires claims to be specific, verifiable, and evidence-backed. Our certificate satisfies that. The roadmap to full ISO 17065 certification begins in Year 2 through a partnership with an accredited DACH certification body.
